# Coauthor Evidence Checklist

Use this checklist before drafting or revising the paper.

## Claim strength

- Can each claim be traced to `evidence-pack/claim-evidence-map.md`?
- Which claims are only observations?
- Which claims can be written as method proposals?
- Which claims must wait for experiments?
- Does the wording distinguish seed examples from empirical evidence?

## Evidence status

- Are illustrative examples clearly labeled as illustrative?
- Are seed cases clearly labeled as seed cases?
- Is any example accidentally written as a real experimental result?
- Does the draft avoid pretending there are human annotation results?
- Does the draft avoid pretending there are multi-model experiments?

## Paper scope

- Do the title and abstract overpromise?
- Does the paper claim the method has been validated before evaluation?
- Does the paper imply failure frequency without data?
- Does the paper default to top-conference standards while ignoring technical report, preprint, or workshop goals?

## Concept integrity

- Does the draft preserve Gate 1: Frame Lock?
- Does the draft preserve Gate 2: Dialogue Boundary Lock?
- Does it avoid turning the project into generic tone optimization?
- Does it avoid turning the project into generic prompt engineering?
- Does it avoid turning the project into a complex multi-agent protocol?
- Does it avoid treating Instruction Drift Guard as a third primary failure family?

## Next action

If any answer is uncertain, weaken the claim, add a TODO, or move the statement to limitations.
